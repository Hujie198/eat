# 家常菜点菜工具 - 产品需求文档

## 1. 项目概述

### 1.1 项目背景
本项目旨在为北方用户打造一个便捷的家常菜点菜工具，帮助用户浏览、选择和记录各类家常菜品，提供菜品做法参考。

### 1.2 产品价值
- 提供丰富的北方菜系菜品选择
- 方便用户快速选择想吃的菜品
- 提供详细的菜品做法和食材清单
- 记录用户的点菜历史

### 1.3 目标用户
北方地区家庭用户，喜欢家常烹饪的人群

---

## 2. 功能需求

### 2.1 导航栏（底部导航）
- 三栏导航：菜单、干饭、我的
- 点击切换页面

### 2.2 菜单页面
#### 2.2.1 一级菜单（菜系分类）
- 河南菜
- 川菜
- 湘菜
- 江西菜
- 山东菜
- 其他北方菜系

#### 2.2.2 二级菜单（菜品列表）
- 每个菜系展示50个常吃的热门菜
- 每个菜品显示：菜名、难度标签、销量
- 每个菜品有添加按钮

#### 2.2.3 菜品详情
- 点击菜品展示详细信息
- 包含：食材清单、烹饪步骤

### 2.3 干饭页面
- 展示已添加的菜品列表
- 显示清除按钮，一键清空列表
- 可查看每个菜品的做法

### 2.4 我的页面
- 记录用户点过的所有菜品（历史记录）
- 显示点菜总次数统计

---

## 3. 页面结构

| 页面 | 功能 | 状态 |
|------|------|------|
| 菜单页 | 菜系选择、菜品列表、销量展示 | 核心功能 |
| 干饭页 | 当前已点菜品、清除按钮 | 核心功能 |
| 我的页 | 点菜历史记录、总次数统计 | 核心功能 |

---

## 4. 数据结构

### 4.1 菜品数据
```json
{
  "id": "菜品ID",
  "name": "菜名",
  "category": "菜系",
  "difficulty": "难度（简单/中等/困难）",
  "sales": "销量",
  "ingredients": ["食材1", "食材2", ...],
  "steps": ["步骤1", "步骤2", ...],
  "image": "图片URL"
}
```

### 4.2 用户数据
```json
{
  "id": "用户ID",
  "cart": ["菜品ID列表"],
  "history": ["菜品ID列表"],
  "totalOrders": "总点菜次数"
}
```

---

## 5. 交互流程

1. 用户进入菜单页
2. 选择菜系（一级菜单）
3. 浏览菜品列表（二级菜单）
4. 点击添加按钮，菜品加入干饭列表
5. 切换到干饭页查看已选菜品
6. 点击菜品查看做法
7. 点击清除按钮清空列表
8. 切换到我的页查看历史记录

---

## 6. 技术要求
- 响应式设计，支持移动端和桌面端
- 数据持久化存储（localStorage）
- 流畅的动画效果
- 美观的UI设计

---

## 7. 非功能性需求
- 页面加载时间 < 2秒
- 支持离线浏览
- 数据本地存储
